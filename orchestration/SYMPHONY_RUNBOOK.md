# Symphony Runbook (Work-Item / Proof-of-Work)

## 1) Work-item intake
- Assign Work Item ID and Contract ID
- Define expected gate path (G1..G5)
- Set owner and escalation priority

## 2) Execution loop
1. Implement smallest verifiable unit
2. Record proof-of-work using template
3. Update gate decision log
4. Trigger escalation if SLA thresholds are exceeded

## 3) Evidence minimum
- Issue link
- PR link
- Verification note/log
- Decision rationale at each gate transition

## 4) Completion criteria
- Gate reaches G5 (Go)
- Evidence set complete and linkable
- Weekly review reflects closure and residual risk
